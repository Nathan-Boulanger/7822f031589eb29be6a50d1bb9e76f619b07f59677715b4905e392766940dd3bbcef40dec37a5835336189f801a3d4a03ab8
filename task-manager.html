CLM Task Manager Service REST

Base URL: https://{host}:{port}/tasks

  • host: required(string)
  • port: required(number)

Task manager service description

Service used to manage tasks, i.e create new tasks, start them on-demand & retrieve their statuses

post /tasks

Create a new task definition, DEFAULT SCHEDULER IS NONE & START THE TASK AT CREATION + ON DEMAND

  • Request
  • Response

Body

Media type: multipart/form-data

Type: object

Properties
  • taskType: required(string)

    Task type. Dictionnary TASK_TYPES

  • ejbJNDI: required(string)

    JNDI of object in charge of execution

  • ejbMethod: required(string)

    Java method executed by the task

  • status: required(string)

    Status type. Dictionnary GENERAL_STATUSES (A/I)

  • name: required(string)

    Name of the task to create

  • scheduleMode: required(string)

    Schedule mode of the task (CronExpression, IntervalPeriod, ManualSetup, LastDayOfMonth)

  • storeIdleTask: (boolean)

    Used to specify if you want to store idle task execution or not (default value is false)

  • description: (string)

    Description of the task

  • nextExecDate: (string)

    Next exec date (specific to ManualSetup)

  • interval: (string)

    Interval of execution (specific to IntervalPeriod)

  • intervalUnit: (string)

    Interval unit of execution (Day, Month... specific to IntervalPeriod)

  • cronExpression: (string)

    CronExpression for the execution (specific to CronExpression)

HTTP status code 201

Succesfully added a new task definition

Headers

  • Location: (string)

    URL of the newly created task definition

HTTP status code 400

List of error codes

Body

Media type: application/json

Type: object

Properties
  • BAD_SCHEDULE_MODE: (string)

    Tried to create a scheduled task without specifying a valid schedule mode

  • NO_NEXT_EXEC_DATE: (string)

    Tried to create a ManualSetup scheduled task without specifying the nextExecDate field

  • NO_INTERVAL_DEFINITION: (string)

    Tried to create a IntervalPeriod scheduled task without specifying either the interval or intervalUnit fields

  • BAD_INTERVAL_UNIT: (string)

    Tried to create a IntervalPeriod scheduled task with a bad interval unit

  • NO_CRON_EXPRESSION: (string)

    Tried to create a CronExpression scheduled task without specifying the cronExpression field

  • VALIDATION_ERROR: (string)

    Constraint violations (required fields cannot be null)

HTTP status code 500

List of error codes

Body

Media type: application/json

Type: object

Properties
  • TASK_NOT_UNIQUE_NAME: (string)

    Tried to create a task with an already existing name

  • INTERNAL_ERROR: (string)

    An internal error occured while trying to create the scheduled task
